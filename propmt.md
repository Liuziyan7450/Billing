# 简记账 App 提示词模板（Prompt Template）

你是一个资深 Android 工程师，请在 **不破坏现有功能** 的前提下，对一个离线记账 App（Jetpack Compose + Room）进行增量开发。

## 项目背景
- 应用名称：简记账（Billing）
- 技术栈：Kotlin / Jetpack Compose / Material 3 / Room / kotlinx.serialization
- 核心页面：
  1. 首页（记录列表 + 快速记账）
  2. 分类页（支出/收入分类与预算）
  3. 统计页（概览/趋势/分类）
- 当前支持：
  - 收入支出分离分类
  - 支出预算与进度
  - 趋势日历
  - 饼图与柱状图
  - JSON 全量导出与合并去重导入

## 需求要求（执行时必须遵守）
1. **离线优先**：不得依赖在线服务。
2. **兼容旧数据**：导入旧版 JSON 不崩溃。
3. **不可破坏导入导出**：
   - 导出应包含 metadata + categories + records；
   - 导入应是“合并去重”，不得先清库覆盖。
4. **UI 交互一致性**：
   - 底部弹层风格统一；
   - 统计页支持滑动切换；
   - 首页/分类/统计支持左右滑动切页。
5. **分类删除规则**：
   - 有关联记录的分类禁止删除，仅提示；
   - 无记录时才允许删除。
6. **代码质量**：
   - 提供明确的状态流和数据流；
   - 尽量避免重复逻辑；
   - 变更后给出可执行验证命令。

## 输出格式要求
- 先给“实现方案概要”。
- 再给“具体改动文件列表”。
- 再给“关键代码片段（仅必要）”。
- 最后给“测试与验证命令 + 结果说明”。

## 验证清单
- [ ] 新增记录后首页即时可见
- [ ] 统计-趋势默认显示当日累计收支
- [ ] 点击同一天可弹出详情 Sheet
- [ ] 饼图点击可显示分类名称
- [ ] 柱状图可显示分类金额对比
- [ ] 导入 JSON 不覆盖本地当日已有数据
- [ ] 重复导入同一备份不会无限重复记录
