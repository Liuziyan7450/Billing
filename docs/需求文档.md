# 简记账 App 需求文档（最新实现版）

## 一、问题与目标

### 已发现问题
1. 记账弹层高度不足，内容显示不完整。
2. 收入与支出错误共用同一批分类。
3. 分类页预设分类重复生成。
4. 统计页概览维度不足（仅支出）。
5. 趋势页日历展示弱，且无法查看单日明细。
6. 分类页筛选按钮无明显高亮，展示形式不够直观。
7. 统计页存在视觉留白。
8. 导入/导出按钮含义不明确。
9. App 图标辨识度差。

### 本次目标
- 修复上述 9 项问题，并保持离线 + JSON 备份能力。

## 二、功能需求

### 1) 记账弹层
- 底部弹层应可完整展示，支持高内容密度布局。
- 使用更高弹层（约 90% 以上屏高）+ 可滚动内容。

### 2) 收入与支出分类隔离
- 支出仅可选择支出分类。
- 收入仅可选择收入分类（预设：工资、红包、其他收入）。
- 记账时切换“收入/支出”后，分类列表联动切换。

### 3) 分类管理页双栏管理 + 去重
- 分类页需有两栏（支出分类 / 收入分类）切换管理。
- 预设分类只初始化一次，不得重复插入。

### 4) 统计页-概览
- 展示以下指标：
  - 当日支出
  - 本月支出
  - 本月收入
  - 本月结余（收入 - 支出）

### 5) 统计页-趋势
- 使用日历插件展示（带星期标题）。
- 日历同时标识“有支出/有收入”的日期（颜色区分）。
- 点击任意日期可查看当日收支明细。

### 6) 统计页-分类
- 时间维度按钮（当日/本周/本月/本年）必须有明确选中高亮。
- 展示方式改为饼图（使用第三方图表插件）。
- 增加收入/支出构成切换。

### 7) 统计页视觉平衡
- 在图表或概览模块补充说明卡片/提示卡，降低大面积空白。

### 8) 导入/导出交互改造
- 将导入/导出整合为一个“数据操作”入口。
- 二级菜单明确“导入备份（JSON）/导出备份（JSON）”。

### 9) APP 图标
- 重绘为自定义 adaptive icon，替换系统默认图标。

## 三、数据与架构要求
- 分类新增类型字段：`EXPENSE` / `INCOME`。
- 支持按分类类型查询。
- 预设分类初始化采用“空库才种子”策略。
- 导入导出 JSON 兼容旧字段，尽量平滑恢复。

## 四、验收标准
1. 弹层内容完整可见，无截断。
2. 收入与支出分类完全隔离。
3. 分类不再重复生成。
4. 概览含支出、收入、结余。
5. 趋势页可看星期、可点日期看明细、可区分收入/支出日期。
6. 分类页按钮有高亮，且展示为饼图。
7. 统计页视觉更均衡。
8. 导入导出入口含义清晰。
9. 新图标生效。
