# 简记账 App 需求文档（本次迭代）

## 1. 背景
当前版本存在以下问题：
1. 收支记录创建后无法删除；
2. 统计页对新增数据展示不明显；
3. 金额输入依赖系统键盘，录入效率低，收入/支出选择不够直观；
4. 整体视觉偏朴素。

## 2. 本次目标
- 提供完整的「记录删除」链路（UI + ViewModel + Repository + DAO）。
- 强化统计页展示，确保新增记录可在统计面板可见。
- 将金额输入改为内置计算器式数字键盘。
- 对收入/支出单选状态提供清晰高亮。
- 升级 UI 风格，使其更贴近 Material 3 的现代视觉。

## 3. 功能需求

### 3.1 记录删除
- 在首页当日记录卡片右侧提供删除按钮。
- 删除后应即时刷新首页列表与趋势统计。
- 数据层需要支持按记录 ID 删除。

### 3.2 统计页展示优化
- 统计页保留「概览 / 趋势 / 分类」三标签：
  - 概览：显示当日开支、当月日均开支；
  - 趋势：按月按日展示支出卡片；
  - 分类：按时间范围（日/周/月/年）展示各分类金额与占比进度。
- 新增记录后，统计数据需自动更新。

### 3.3 金额输入体验
- 记账底部面板金额输入改为“自绘数字键盘”：
  - 包含 0-9、小数点、清空、退格；
  - 不弹系统输入法。
- 金额显示区使用大号金额文本，保持可读性。

### 3.4 收入/支出高亮
- 采用可选中 Chip（FilterChip）表达“支出/收入”单选。
- 选中态必须与未选中态有明确对比。

### 3.5 视觉升级
- 首页增加渐变背景与摘要卡片（今日收入/支出）。
- 各模块卡片增强容器色与层次。
- 全局主题使用更有品牌感的暖色系 Material 3 色板。

## 4. 非功能要求
- 继续保持完全离线可用。
- 现有 JSON 导入/导出功能保持兼容。
- 不引入必须联网的第三方服务。

## 5. 验收标准
1. 新增任意记录后，可在首页列表看到该记录，并可删除；
2. 删除记录后，首页与统计页数据同步变化；
3. 录入金额仅通过内置数字键盘完成，不依赖系统输入法；
4. 收入/支出切换有清晰高亮；
5. UI 视觉相比旧版更有层次感与可读性。
